# optional-demo

Java の `java.util.Optional` の使い方を学ぶための **個人学習用プロジェクト** です。

> [!NOTE]
> このリポジトリは学習・検証を目的とした個人プロジェクトです。
> 実務での利用や本番環境での動作を想定したものではありません。

## 概要

ユーザー情報の検索を題材に、`Optional` から値を取り出す代表的な3つの方法を比較できるサンプルコードです。
「値が存在する場合」と「存在しない場合」の両方を実行し、それぞれの挙動の違いを確認できます。

## 実行環境

- OpenJDK 21

## ファイル構成

| ファイル | 役割 |
| --- | --- |
| `Main.java` | エントリーポイント。`Optional` の3つの取り出し方を実演する |
| `User.java` | ユーザーを表す record（`id` / `name` / `birthday`） |
| `UserRepository.java` | ユーザー検索を模した擬似リポジトリ。ID:1 のみ存在する |

## 学べる内容

`Optional` から値を取り出す（または変換する）3つのパターンを扱っています。

| メソッド | 値がある時 | 値が無い時 |
| --- | --- | --- |
| `get()` | 中身を返す | `NoSuchElementException` をスローする |
| `orElseThrow(...)` | 中身を返す | 指定した任意の例外をスローする |
| `map(...)` | 中身を変換して `Optional` で返す | 何もせず空の `Optional` を返す |

あわせて、`map` や `orElseThrow` に渡す **ラムダ式** の書き方も確認できます。

## 実行方法

```bash
# コンパイル
javac *.java

# 実行
java Main
```

> [!TIP]
> `UserRepository#findById` は ID:1 のユーザーのみを返します。
> それ以外の ID を指定すると空の `Optional` が返るため、「値が無い場合」の挙動を確認できます。
